# Tutorial Prompt Template

把下面这段作为写教程时的默认提示词模板，替换其中的主题即可。

```text
请按我的教程写作风格，帮我写一篇面向组内新人的教程。

主题：<在这里填写主题>
目标读者：刚入组、基础不强但需要尽快上手的同学
文章目标：让读者在最短时间内具备最小可用能力

写作要求：
1. 语气像有经验的师兄带新人，不要居高临下，也不要过度安慰
2. 结论先行，少废话，优先讲“先做什么最有用”
3. 文章不是百科，不追求全面，优先保证实用
4. 如果适合，按“基础 / 进阶”或者“必须掌握 / 进阶掌握”组织
5. 所有命令示例尽量用这种格式：

```bash
pwd      # 显示当前目录
ls -l    # 查看详细文件信息
cd ..    # 返回上一级目录
```

6. 配置文件示例也尽量逐行解释
7. 多讲科研场景里的实际用途，少讲抽象定义
8. 补充常见坑、最小检查清单、学完后能做什么
9. 如果有旧文章内容可整合，优先整合最有价值的部分，去掉重复内容
10. 标题层级清晰，方便网页自动目录显示

默认结构：
- 标题
- 目标
- 前言 / 为什么要学
- 正文主体
- 常见坑
- 入门检查清单
- 下一步建议

请直接输出可发布的 mdx 正文，不要输出写作说明。
```

## 极简版

如果只是想快速开写，可以直接用这段：

```text
请按我一贯的教程风格，帮我写一篇面向组内新人的教程：

- 结论先行
- 语气像师兄带新人，明确但不压人
- 优先讲最小可用路径
- 命令示例逐行加 # 中文注释
- 多用科研场景例子
- 结尾加常见坑和入门检查清单

主题：<在这里填写主题>

请直接输出可发布的 mdx 正文。
```
