# AGENTS

## 组内教程写作原则

写组内教程的第一目标，不是把一个主题讲得像教科书一样完整，也不是追求面面俱到。

第一要义是：

- 开阔眼界
- 降低起步门槛
- 让读者先知道“有这个东西存在”
- 让读者知道它大概是做什么的
- 让读者知道什么时候可能会用到它

核心判断是：

在 LLM 时代，细节知识本身并不稀缺，很多具体命令、参数、报错、实现步骤，后续都可以再查、再问、再补。

真正稀缺的是：

- 不知道某个工具、方法或工作流的存在
- 不知道自己遇到的问题可以往哪个方向找
- 不知道下一步该学什么、问什么、试什么

所以组内教程的价值，不是替代所有资料，而是完成“破冰”和“开路”。

具体写法上，应优先做到：

1. 先告诉读者这是什么
2. 再告诉读者它有什么用
3. 再告诉读者在什么场景下会遇到它
4. 给出最小可用例子
5. 让读者知道后续可以继续往哪里查、问、扩展

不必默认把所有细节一次讲完。

如果一篇文章过于细致，容易带来两个问题：

- 新人一开始读不完，也抓不住重点
- 读者会误以为“必须全部掌握后才能开始动手”

更合适的标准是：

- 先把门打开
- 先让人有全局感
- 先让人迈出第一步

万事开头难。

因此，组内教程首先要解决的，不是“知识是否足够完整”，而是“新人能不能顺利开始”。
