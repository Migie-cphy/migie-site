# 3.4 Vaspsol++

这一篇不追求把 Vaspsol++ 所有参数一次讲全。

它的目标只有一个：

> 让你第一次接触 Vaspsol++ 时，先知道它是什么、什么时候该用、最小怎么跑起来。

如果后面你要进一步调参、做更严格的电势标定，再回头看官方文档和论文会更合适。

## 1. Vaspsol++ 是什么

Vaspsol++ 是一个在 VASP 中加入连续介质流体模型的框架。

你可以先把它粗略理解成：

- 普通 VASP：默认是真空里的电子结构计算
- `VASPsol`：给 VASP 加上线性局域的隐式溶剂模型
- `Vaspsol++`：在 `VASPsol` 基础上进一步支持更复杂的非线性、非局域电解液模型

它特别适合下面这些场景：

- 想在 VASP 中考虑隐式溶剂
- 想描述带电电极附近的电解液响应
- 想做恒电势而不是恒电子数的计算

## 2. 它相比原始 VASPsol 多了什么

根据你给的文档，Vaspsol++ 最核心的增强可以先记成这几条：

- 支持非线性介电响应和非线性离子响应
- 支持非局域 cavity 定义，减少电解液“漏进”不合理小区域的问题
- 可以区分 dielectric cavity 和 ionic cavity
- 支持用 Newton 方法求解 nonlinear Poisson-Boltzmann 方程
- 支持恒电势计算，也就是 grand canonical DFT

如果你现在只是刚开始接触，先建立一个最重要的判断就够了：

> Vaspsol++ 不是普通 VASP 的“随便加一个修正项”，而是把溶剂和电解液响应真正纳入了自洽计算。

## 3. 什么时候优先考虑 Vaspsol++

比较典型的情况是：

- 研究电催化或电化学界面
- 体系有净电荷，真空计算很难直接解释
- 你关心的是电势控制下的行为，而不是固定电子数下的行为

如果你只是做最普通的中性 slab 吸附、结构优化、能带或 DOS，且不想引入溶剂和电解液，那么普通 VASP 往往就够了。

## 4. 安装上最关键的现实问题

Vaspsol++ 不是简单地把一个 `INCAR` 参数打开就能用。

你需要先把它编译进 VASP。

根据你给的文档，安装上最重要的几点是：

1. `solvation.F` 需要替换进 VASP 源码
2. 还需要一个和 VASP 版本匹配的 patch 文件
3. 这个 patch 文件是版本相关的，需要联系作者获取
4. 目前文档明确提到支持的版本包括：
   - `VASP 5.4.4`
   - `VASP 6.3.2`
   - 以上版本都分带 `VTST` 和不带 `VTST` 两种 patch

所以你第一次排查时，最先要确认的不是 `INCAR`，而是：

- 你的 VASP 版本对不对
- patch 是否对应你的版本
- patch 是否正确打上
- 最终编译出的 VASP 是否真的包含了 Vaspsol++

## 5. 最小安装流程

如果你已经有可编译的 VASP 源码目录 `<VASP_SRC>`，一个最小流程可以理解成：

1. 下载 Vaspsol++ release 的源码包并解压，得到 `<VASPSOL_SRC>`
2. 用 `<VASPSOL_SRC>/src/solvation.F` 替换 `<VASP_SRC>/src/solvation.F`
3. 选择和你 VASP 版本匹配的 patch 文件
4. 在 `<VASP_SRC>` 里执行 patch
5. 按平时方式重新编译 VASP

文档里对应的 patch 命令是：

```bash
patch -p1 < <patch_file>
```

如果你在用 `VTST`，要选带 `vtst` 的 patch；否则选普通 patch。

## 6. 第一次上手最小要知道的三个开关

如果只从 `INCAR` 看，最重要的三个开关是：

```text
LSOL
ISOL
LSOL_SCF
```

可以先这样记：

- `LSOL = .TRUE.`：打开溶剂模型
- `ISOL = 1`：原始 VASPsol 的线性局域模型
- `ISOL = 2`：Vaspsol++ 的非线性非局域模型
- `LSOL_SCF = .TRUE.`：把溶剂模型纳入 SCF 自洽

如果你是第一次真的想用 Vaspsol++，最关键的一句就是：

> 要用 Vaspsol++ 的核心模型，`ISOL` 必须设成 `2`。

## 7. 一个最小的 Vaspsol++ `INCAR`

如果你的目标只是“先跑起来一个最小的水溶液电解液模型”，可以从下面这个最小例子开始：

```text
LSOL = .TRUE.
LSOL_SCF = .TRUE.
ISOL = 2

C_MOLAR = 1.0
R_ION = 4.0
EFERMI_ref = -4.57
```

你可以先把它理解成：

- `LSOL = .TRUE.`：打开隐式溶剂
- `LSOL_SCF = .TRUE.`：让溶剂响应进入自洽过程
- `ISOL = 2`：使用 Vaspsol++ 的 nonlinear + nonlocal 模型
- `C_MOLAR = 1.0`：电解液浓度设为 `1 mol/L`
- `R_ION = 4.0`：离子半径设为 `4 Å`
- `EFERMI_ref = -4.57`：做恒电势计算时给定电子化学势

## 8. 这个最小例子为什么已经够你上手

这是因为文档明确说了：

- 默认参数已经对应 `298 K` 的纯水
- 真正要补充指定的，通常就是电解液浓度 `C_MOLAR`
- 以及离子半径 `R_ION`

如果你不设 `R_ION`，它会默认等于 `R_SOLV`，而文档特别提醒这通常偏小，不一定合理。

所以第一次上手时，一个很实用的最小原则是：

1. 先用默认水参数
2. 明确写出 `C_MOLAR`
3. 明确写出 `R_ION`
4. 如果做恒电势，再写 `EFERMI_ref`

## 9. 恒电势计算最先要知道什么

Vaspsol++ 支持恒电势计算，也就是改变电子数而不是固定电子数。

在输入上，最核心的参数就是：

```text
EFERMI_ref
```

可以先这样理解：

- `EFERMI_ref = 0`：默认，不做恒电势
- `EFERMI_ref < 0`：执行恒电势计算

但这里有一个非常重要的前提：

> 只有当电解液里存在 ionic screening 时，恒电势才有明确参考意义。

所以文档特别强调：

- `C_MOLAR > 0` 时，Fermi level 才能相对于真空被合理定义

如果没有离子 screening，只写 `EFERMI_ref` 并不够。

## 10. `EFERMI_ref = -4.57 eV` 是什么含义

在你给的文档里，这个值对应的是：

- 默认水参数
- `BEEF-vdW` 泛函
- 作者给出的 `SHE` 标定结果

所以一定不要把这个数当作“任何体系都通用”的常数。

更准确地说，它只是：

> 在一套特定泛函和溶剂参数下，作者给出的一个示例值。

如果你以后做正式工作，应该先搞清楚：

- 你用的是什么泛函
- 你的溶剂参数是什么
- 你的 `SHE` 标定值到底是多少

## 11. 第一次上手推荐的策略

不要一上来就直接做恒电势大体系。

更稳妥的顺序通常是：

1. 先确认 Vaspsol++ 版本已经正确编译进 VASP
2. 先做一个普通 `LSOL = .TRUE., ISOL = 2` 的静态计算
3. 再加入 `C_MOLAR` 和 `R_ION`
4. 最后再尝试 `EFERMI_ref`

这样如果出问题，你更容易判断是：

- 编译问题
- 输入参数问题
- 恒电势设置问题

## 12. 常见参数按“第一次上手”怎么分层

第一次不需要把所有参数都背下来。

更适合的分层是：

### 第一层：必须知道

- `LSOL`
- `ISOL`
- `LSOL_SCF`
- `C_MOLAR`
- `R_ION`
- `EFERMI_ref`

### 第二层：知道它们存在

- `LNLDIEL`
- `LNLION`
- `R_SOLV`
- `R_DIEL`
- `NC_K`
- `SIGMA`
- `TAU`
- `EB_K`

### 第三层：以后再细调

- `A_K`
- `R_CAV`
- `R_B`
- `ZION`
- `D_ION`
- `EFERMI_tol`
- `capacitance_init`

## 13. 输出时最应该看什么

根据文档，第一次上手最值得关注的输出量是：

- 自由能
- Fermi level
- `E0`

这里最重要的直觉是：

- 普通情况下，VASP 主程序打印的 free energy 已经包含所需溶剂修正
- 恒电势计算时，打印的是 grand canonical potential `Omega`
- `E0` 也已经包含了相应的溶剂修正

所以新手最容易做错的一件事是：

> 以为还要像旧版 VASPsol 一样，自己再额外手动补一堆修正。

根据你给的文档，Vaspsol++ 在有 ionic screening 的情况下，free energy 和 Fermi level 的参考已经处理好了，不需要再额外补那些旧修正。

## 14. 额外输出文件怎么看

如果你在 `INCAR` 里写：

```text
LVHAR = .TRUE.
```

或者：

```text
LVTOT = .TRUE.
```

那么 Vaspsol++ 还会写出一些额外文件。

第一次上手时，最值得先知道的是这些名字各自大概代表什么：

- `PHI`：电势
- `PHI_SOLV`：溶剂贡献的电势
- `VSOLV`：溶剂相关的 KS 势修正
- `RHOB`：束缚电荷密度
- `RHOION`：电解液离子电荷密度

如果是 `ISOL = 2`，还会看到：

- `SVDW`
- `SSOLV`
- `SION`
- `SDIEL`

对第一次上手的人来说，这几个名字最重要的意义是：

- `SVDW`：和 vdW cavity 有关
- `SSOLV`：溶剂 cavity
- `SION`：离子 cavity
- `SDIEL`：介电 cavity

也就是说，Vaspsol++ 确实在区分不同的 cavity，而不是只用一个简单表面把所有响应都包起来。

## 15. 一个最小工作流

如果把第一次上手压缩成最短流程，大致可以是：

1. 准备一个你已经能正常跑的 VASP slab 体系
2. 确认 Vaspsol++ 已编译进可执行文件
3. 先加：

```text
LSOL = .TRUE.
ISOL = 2
LSOL_SCF = .TRUE.
```

4. 再加：

```text
C_MOLAR = 1.0
R_ION = 4.0
```

5. 如果要恒电势，再加：

```text
EFERMI_ref = ...
```

6. 跑完后先看：
   - 是否正常收敛
   - Fermi level 是否合理
   - free energy / `E0` 是否正常
   - cavity 和 ionic response 相关文件是否输出

## 16. 第一次上手最常见的坑

根据这份文档，最容易踩的坑通常是：

- 以为 `ISOL = 1` 也是 Vaspsol++ 的完整模型
- 忘了 `ISOL = 2`
- 不写 `C_MOLAR` 却想做恒电势
- 不写 `R_ION`，直接用默认值
- 直接照抄 `EFERMI_ref = -4.57`，却没意识到它依赖泛函和参数
- patch 文件和 VASP 版本不匹配

所以第一次出问题时，优先检查这几件事，通常比盲目调很多小参数更有效。
