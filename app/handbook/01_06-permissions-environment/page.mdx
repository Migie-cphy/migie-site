# 1.6 权限和环境变量

## 1. 权限到底在控制什么

在多人共用的服务器上，系统要决定“谁可以对什么东西做什么事”。

放到文件上，最常见的就是三类权限：

- 读 `read`
- 写 `write`
- 执行 `execute`

```bash
ls -l run.sh # 查看文件权限
```

输出通常会长成这样：

```text
-rwxr-xr-- 1 user group 1234 Apr 6 10:00 run.sh
```

可以先这样理解：

- 第一段 `rwx`：文件所有者
- 第二段 `r-x`：同组用户
- 第三段 `r--`：其他用户

## 2. 数字权限怎么看

```bash
chmod 755 run.sh   # 给脚本设置常见的可执行权限
chmod 644 data.txt # 给普通文本文件设置常见权限
```

这里的数字不是硬背出来的，而是二进制位的压缩写法：

```text
r = 100₂ = 4
w = 010₂ = 2
x = 001₂ = 1
```

所以：

```text
7 = rwx = 111₂
6 = rw- = 110₂
5 = r-x = 101₂
4 = r-- = 100₂
```

也就是说：

```text
755 -> rwx r-x r-x
644 -> rw- r-- r--
700 -> rwx --- ---
```

重点不是背 `755` 和 `644`，而是以后看到任意一组数字权限时，你能自己拆开。

## 3. 最常见的权限问题

新手最常遇到的问题不是“文件不能看”，而是“脚本不能执行”。

```bash
chmod +x run.sh # 给脚本增加可执行权限
./run.sh        # 直接执行脚本
```

如果一个脚本没执行权限，先看 `ls -l`，再决定是不是要 `chmod +x`。

## 4. 环境变量是什么

很多“命令找不到”的问题，本质上不是程序坏了，而是环境变量没配对。

```bash
echo $PATH               # 查看 PATH
which python             # 看 python 命令实际来自哪里
which jdftx              # 看 jdftx 命令实际来自哪里
export OMP_NUM_THREADS=4 # 给当前 shell 设置环境变量
```
