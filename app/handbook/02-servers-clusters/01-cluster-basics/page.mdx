# 2.1 集群基础与使用边界

很多新手第一次用服务器时，会把它想成一台“更强的电脑”。

但集群通常不是这样工作的。

你需要先知道几个最基础的角色分工。

## 1. login node 和 compute node

- `login node`：登录、改文件、提交任务、看日志
- `compute node`：真正执行计算

所以一般不应该在登录节点上长期直接跑大任务。

更常见的正确姿势是：

```bash
ssh alias
cd ~/work/project-a
sbatch run.slurm
```

也就是登录节点负责“组织任务”，计算节点负责“执行任务”。

## 2. `home`、`work`、`scratch`

不同机器命名不完全一样，但常见思路是：

- `home`：个人配置和小文件，容量通常不大
- `work`：项目目录和常用计算目录
- `scratch`：大规模临时数据，读写快，但可能会定期清理

所以不要默认把所有结果都长期堆在同一个地方。

## 3. 队列系统为什么存在

多人共用算力时，系统需要回答：

- 谁先跑
- 能用多少核
- 能用多久
- 跑在哪台机器上

这就是 `SLURM` 这类队列系统在做的事。

## 4. 新手最容易踩的坑

- 在登录节点直接跑长时间大任务
- 不知道文件系统会清理，结果把重要数据只放在 `scratch`
- 申请资源和实际程序设置不一致
- 任务挂了以后只盯着脚本，不先看日志和队列状态

## 5. 最小使用边界

如果你暂时不确定一件事该不该做，先遵循这个保守原则：

- 改文件、传文件、查日志：登录节点
- 真正的大计算：通过队列系统提交
- 重要结果：及时整理和备份
