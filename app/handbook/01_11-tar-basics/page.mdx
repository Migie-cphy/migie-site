# 1.11 压缩、解压与 tar

目标：先掌握最常见的打包与解压命令，再知道 `pigz` 是什么、什么时候值得用。

## 1. 最小用法

```bash
tar -czvf dir.tar.gz dir # 将目录压缩成 tar.gz
tar -xzvf dir.tar.gz     # 解压 tar.gz 文件
```

如果你现在完全是新手，先把这两条记住就够了。

## 2. 什么时候会用到

高频场景包括：

- 结果目录太多，需要打包备份
- 需要把一整个目录传给别人
- 下载到一个压缩包，需要先解压再看内容

## 3. 什么是 `pigz`

`pigz` 可以简单理解成“并行版的 gzip”。

当目录很大、文件很多时，它通常会比普通 `gzip` 更快。

```bash
tar -cf - results | pigz > results.tar.gz # 使用 pigz 并行压缩目录
pigz -d -c results.tar.gz | tar -xf -     # 解压 pigz 压缩结果
```

你现在不用一上来就把这两条背下来。更重要的是先知道：

- `tar` 是最基础、最常见的打包解包工具
- `pigz` 是在数据更大时值得考虑的加速版本

## 4. 这一节最少要做到什么

1. 会压缩一个目录
2. 会解压一个 `tar.gz`
3. 知道 `pigz` 是并行压缩工具，目录很大时可以再回来查
