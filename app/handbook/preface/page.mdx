# 前言

这一套手册总结了我硕博4年来我自身的一些技能。
需要说明的是，它并不试图覆盖所有内容，也不能替代系统性的教材、官方文档或参考手册。
更合适的理解方式是：它是一张面向初学者的“地图”。
它的作用，是先帮助读者建立基本认知——知道有哪些工具、
它们分别能解决什么问题、在什么场景下值得使用——然后再带你完成最初的上手过程，实现从0 到 1 的过渡。

因此，这份手册并不预设每个人都需要掌握其中的全部内容。
不同的人有不同的需求、任务和偏好，能够根据自身情况选择合适的部分来学习和使用，就已经足够。

从个人风格上说，我在组内可能算是相对“另类”的一类人。
我长期偏好 CLI 与 coding：凡是可以通过命令行或代码实现的事情，我通常不会优先使用图形界面。
原因并不在于某种形式上的“极客偏好”，而在于我认为 CLI 和 coding 具有更强的可重复性与可积累性，
更适合作为一种长期建设和持续优化的工作方式。
当然，这样的偏好并不适用于所有人。
过去和组内同学交流这些内容时，确实有人会觉得它们“太难”了；
对不少人而言，最重要的是先掌握最低成本、足够可用的技能，而不是追求极致效率。
这完全可以理解，也没有高下之分。

因此，这份手册中介绍的命令、方法和工作流都不是唯一答案，它们更多只是基于我的个人 taste 所做出的选择。
我更希望这份手册能起到一个起点的作用：帮助大家从解决实际问题出发，逐渐形成适合自己的工具观与工作流。

为了目录长期更稳定，读者一眼能看出每部分在讲什么，这套手册现在按主题分类组织。
大部分内容都是基础，我想大部分人都是值得一看的，少部分是满足组内小部分需求，有需要再看。

在 LLM 时代，很多具体细节后面都可以再查、再问、再补。真正容易卡住人的，往往不是“资料不够多”，而是：

- 不知道有这个东西
- 不知道自己遇到的问题该往哪个方向找
- 不知道下一步该学什么、问什么、试什么

Claude、ChatGPT、Gemini...是人类有史以来最好的老师，对于后续的内容，如果你感兴趣或者需要深入就去找他们吧！！

另外，今天 agents 在这些事情上的效率已经远远超过了人类，在使用这些工具的同时，
希望你可以知道它在干什么，它会犯错，不要被它诈骗了，我想这也是对我们提出了更高的要求。
