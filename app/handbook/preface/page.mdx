# 前言

这一套手册总结了我硕博4年来我自身的一些技能。
它的目标不是把所有东西一次讲全，也不是替代教材、官方文档和参考手册。
更准确地说，这里更像是一张地图：先帮你知道有什么工具、它大概是做什么的、什么时候可能会用到，然后再带你迈出第一步，实现从“0”到“1”。
同时，并不要求你把所有的东西都学会，各取所需吧！
在我的印象中，我在组内应该属于比较“另类”。
我是偏执的 CLI、coding 拥护者，能写代码解决的事情，我一定不会使用图形化解决。
因为 CLI 和 coding 可重复可积累，是一件长期主义的事。
过去，跟组内的同学分享这些，部分人会觉得太“难”了，他们只需要最小的可使用的技能即可，而不追求高效。这也很好～
另外，所用的命令和方法并不是唯一，是基于我的 taste 而写的，鼓励大家基于解决问题出发，培养自己的 taste 建设优化自己的工作流。

为了目录长期更稳定，读者一眼能看出每部分在讲什么，这套手册现在按主题分类组织。
大部分内容都是基础，我想大部分人都是值得一看的，少部分是满足组内小部分需求，有需要再看。

在 LLM 时代，很多具体细节后面都可以再查、再问、再补。真正容易卡住人的，往往不是“资料不够多”，而是：

- 不知道有这个东西
- 不知道自己遇到的问题该往哪个方向找
- 不知道下一步该学什么、问什么、试什么

Claude、ChatGPT、Gemini...是人类有史以来最好的老师，对于后续的内容，如果你感兴趣或者需要深入就去找他们吧！！

另外，今天 agents 在这些事情上的效率已经远远超过了人类，在使用这些工具的同时，
希望你可以知道它在干什么，它会犯错，不要被它诈骗了，我想这也是对我们提出了更高的要求。
