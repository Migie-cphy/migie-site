# 1.4 查看文件与日志

## 1. 最常见的几个命令

| 命令 | 全称 | 主要作用 |
| --- | --- | --- |
| `cat` | concatenate | 直接输出文件内容，适合小文件 |
| `less` | less | 分页查看文件，适合大文件 |
| `head` | head | 查看文件开头几行 |
| `tail` | tail | 查看文件最后几行，或持续追踪新输出 |

例如：

```bash
cat INCAR          # 直接输出所有文件内容
head -n 20 OUTCAR  # 查看前 20 行 默认是 10 行
tail -n 20 OUTCAR  # 查看后 20 行 默认是 10 行
less OUTCAR        # 翻页查看大文件
```

简单理解：

- 小文件用 `cat`
- 大文件优先用 `less`
- 看文件开头用 `head`
- 看文件结尾和最新输出用 `tail`

## 3. 看日志时的基本思路

看输出文件时，最有效的方式通常不是从头读到尾，而是先看关键位置。

一个最基本的工作流是：

```bash
head -n 20 OUTCAR  # 先看开头是否正常
tail -n 50 OUTCAR  # 再看最后几十行有没有报错或异常结束
less OUTCAR        # 需要时再翻页细看
```

如果你要持续盯着日志变化，可以用：

```bash
tail -f job.out # 实时追踪日志新输出
```

## 4. 常见误区

### 用 `cat` 打开超大文件

大文件优先用：

```bash
less file         # 翻页查看大文件
tail -n 50 file   # 看最后 50 行
```

### 一上来就从第一行开始硬读

很多时候，真正有信息量的位置是：

- 文件开头
- 文件最后几十行
- 正在持续刷新的最新输出
