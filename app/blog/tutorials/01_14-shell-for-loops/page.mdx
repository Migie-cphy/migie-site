# 1.14 批处理脚本：for

目标：看到重复劳动时，开始有“这件事也许可以自动化”的意识。

## 1. 最小例子

```bash
for i in {01..10}; do
  echo $i # 逐个输出 01 到 10
done      # 循环结束
```

## 2. 科研里更常见的写法

```bash
for i in *; do
  cp $i/CONTCAR zzz/$i.vasp # 复制每个目录下的 CONTCAR 并重命名
done                        # 循环结束
```

```bash
for i in *.vasp; do
  mkdir ${i/.vasp}        # 用文件名创建同名目录
  mv $i ${i/.vasp}/POSCAR # 把文件移动进去并改名为 POSCAR
done                      # 循环结束
```

## 3. 这节真正重要的点

重点不是背语法，而是慢慢形成一个判断：

- 重复劳动通常都能写成循环
- 目录结构一旦统一，就很容易自动化

## 4. 这一节最少要做到什么

1. 知道 `for ... do ... done` 是循环结构
2. 看懂遍历文件或目录的最简单写法
3. 开始对“批处理”建立感觉
