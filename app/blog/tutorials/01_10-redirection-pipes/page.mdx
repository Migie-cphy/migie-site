# 1.10 重定向和管道

目标：开始理解命令行为什么能把很多小命令串起来完成工作。

## 1. 重定向

```bash
python script.py > output.log  # 覆盖写入日志
python script.py >> output.log # 追加写入日志
```

- `>`：覆盖写入
- `>>`：追加写入

这两个一定要分清，不然日志很容易被你自己覆盖掉。

## 2. 管道

```bash
tail -n 100 job.out | grep energy # 查看最后 100 行并筛选 energy
cat job.out | grep error          # 输出文件内容并筛选 error
```

可以先把 `|` 理解成“把前一个命令的输出交给后一个命令继续处理”。

## 3. 为什么这一节重要

很多时候你会发现，一个命令本身不复杂，但几个命令串起来就很好用。

比如：

```bash
grep error job.out # 直接在文件里搜索 error
```

和：

```bash
tail -n 100 job.out | grep error # 只在最后 100 行里搜 error
```

这两个命令解决的问题就已经不一样了。

## 4. 这一节最少要做到什么

1. 分清 `>` 和 `>>`
2. 知道 `|` 是把前一个命令的输出交给下一个命令
3. 能看懂最基础的管道命令
