# Linux 与命令行：入门必备

目标：帮助你尽快具备独立使用命令行控制 Linux 服务器的基础能力。

## 前言

### 为什么学习命令行？

1. Linux 服务器主要就是通过命令行使用的。
2. 所有的科学计算软件 (VASP, LAMMPS...) 都需要通过命令行实现。

### CLI 和 GUI

说到电脑，很多同学更熟悉的是图形界面（Graphical User Interface, GUI），比如点开文件夹、双击运行程序、通过窗口切换任务。
但在科研计算环境里，命令行（Command Line Interface, CLI）通常更重要。区别如下：

| 交互方式 | 典型操作 | 优点 | 局限 |
| --- | --- | --- | --- |
| GUI | 鼠标点点点，在对应的地方输入对应的东西 | 直观、上手快 | 不适合远程、大批量、自动化任务 |
| CLI | 在终端输入文本命令 | 逻辑简单，高效 | 不够直观，初学时有门槛 |

对科学计算来说，CLI 更重要，不是因为它“看起来更专业”，而是因为它更适合下面这些场景：

- 远程操作服务器
- 批量处理文件
- 自动化重复任务
- 记录和复现完整流程


这一篇只保留最核心的入门内容：先让你能正常登录服务器、整理文件、查看输出、完成基础传输。

如果你完全没用过 Linux，我建议你先把这一篇过一遍，并亲自操作。不要试图一口气背下来，重点是先知道这些命令分别在干什么。

## 第一部分：基础

### 1. 登录服务器

第一件事是登录服务器。

Windows 用户可以用 PowerShell，或者更推荐直接装 `WSL`。如果你更习惯图形化终端工具，也可以先用 [`Xshell`](https://www.netsarang.com/en/xshell/) 、[`MobaXterm`](https://mobaxterm.mobatek.net/) 这类软件连接服务器。macOS 用户直接打开终端即可。

#### 密码连接

```bash
ssh -p port username@hostname # 用指定端口连接远程服务器

# -p 后面是端口号，如果端口是默认值 22 可以不写
# username 是用户名
# hostname 一般是 IP 或者域名
```

回车执行命令后，一般是让你输入密码，输入即可登录成功。

注意：终端内的密码不会明文显示，即输入是没反应的，不是卡住了。

#### 密钥连接

如果你经常登录同一台服务器，建议配置 `~/.ssh/config` 并且使用密钥连接：

```ssh-config
Host alias                       # 给这台服务器起一个本地别名
    HostName hostname            # 服务器地址，通常是 IP 或域名
    User username                # 登录用户名
    Port port                    # SSH 端口
    IdentityFile path/to/private/key # 私钥路径
```

这样以后只要：

```bash
ssh alias # 使用别名连接服务器
```

这样做的好处在于以后：

- `ssh`
- `scp`
- `rsync`
- `sftp`
- `...`

这些和服务器通信的命令也都可以直接复用这个别名，不用每次重新记 `IP`、用户名和端口。

#### 密钥配置

刚刚解决了每次重复输入`IP`、用户名和端口的问题，我们现在解决每次都要输入密码的问题。

#### 使用 `ssh-keygen` 生成密钥对

```bash
ssh-keygen # 生成一对 SSH 密钥
# 一直按回车就好了，感兴趣的可以看看在干嘛
```

终端会提示你保存路径和密码。默认一般是：

```text
~/.ssh/id_rsa
~/.ssh/id_rsa.pub
```

其中：

- `id_rsa` 是私钥
- `id_rsa.pub` 是公钥

`IdentityFile` 对应的就是私钥路径。默认情况下通常就是 `~/.ssh/id_rsa`。

#### 使用 `ssh-copy-id` 复制公钥到服务器

如果本地有 `ssh-copy-id`，这是最省事的做法：

```bash
ssh-copy-id alias # 将本地公钥复制到服务器 authorized_keys
```

它会把你的公钥自动追加到服务器的 `~/.ssh/authorized_keys` 中。

#### 手动复制公钥

如果没有 `ssh-copy-id`，也可以手动复制 `id_rsa.pub` 里的内容，粘贴到服务器的：

```text
~/.ssh/authorized_keys
```

如果这个文件里已经有内容，就另起一行追加，不要覆盖原来的内容。

配置完成后，`ssh alias` 一般就可以直接通过密钥连接，不用每次重新输入密码。

### 2. 文件和目录操作

这些命令是日常频率最高的一组。这里只给出简单的例子，如果感兴趣可以问 LLM Model，应该会更有收获。

这里是文件和目录操作最经典的一些命令：

| 命令 | 全称 | 主要作用 |
| --- | --- | --- |
| `pwd` | print working directory | 显示当前所在目录 |
| `ls` | list | 列出当前目录下的文件和文件夹 |
| `cd` | change directory | 切换目录 |
| `mkdir` | make directory | 创建目录 |
| `cp` | copy | 复制文件或目录 |
| `mv` | move | 移动文件，或者给文件重命名 |
| `rm` | remove | 删除文件或目录 |

可以看到其实就是英文的首字母缩写，然后通过添加参数实现各种功能。

最常见的用法：

```bash
pwd                        # 显示当前所在目录
ls                         # 列出当前目录下的文件和文件夹
ls -l                      # 以详细格式显示文件信息
ls -a                      # 显示隐藏文件
ls -t                      # 按修改时间排序，新的排前面
ls -r                      # 反序显示
ls -R                      # 递归显示子目录内容

cd ..                      # 切换到上一级目录
cd -                       # 切换回上一次所在的目录
cd ~                       # 切换到用户家目录
cd /                       # 回到根目录
cd relative/path           # 进入相对路径
cd /absolute/path          # 进入绝对路径

mkdir test                 # 创建一个名为 test 的目录
mkdir -p project/run1/logs # 递归创建多级目录

cp input.in backup.in      # 复制文件并另存为 backup.in
cp -r old_dir new_dir      # 递归复制整个目录

mv file1 file2             # 将 file1 重命名为 file2
mv *.txt results/          # 将当前目录下所有 txt 文件移动到 results 目录
```

其中 `ls -t` 很有用。文件多的时候，你很容易忘记最近动过哪个目录，按修改时间排序通常能快速定位。

#### `Tab` 补全

这也是非常值得尽早养成的习惯。

在终端里输入一部分文件名、目录名或者命令后，按 `Tab` 键，shell 会尝试帮你自动补全。

比如：

```bash
cd proj<Tab>          # 自动补全目录名
cp INC<Tab> backup    # 自动补全文件名
ssh gp<Tab>           # 自动补全别名或命令
```

它的好处很直接：

- 少打字
- 少拼写错误
- 文件路径很长时更省事

但也要注意：`Tab` 补全虽然方便，不代表它补出来的就一定是你真正想操作的目标。尤其是在执行 `rm`、`mv` 这类命令时，补全之后还是建议先看一眼，避免选错对象。

`rm` (remove) 是最需要小心的命令之一。

```bash
rm file    # 删除一个文件
rm -r dir  # 递归删除目录
rm -rf dir # 强制递归删除目录，危险
```
`rm` 没有回收站没有废纸篓，删了就真没了，几乎不可能恢复，就算能也非常费劲。

我想几乎所有人都会遇上误删的经历。所以这里给一个很实际的建议：

- 尽量不要形成无脑 `rm -rf` 的习惯
- 删除前先 `pwd`
- 再 `ls`
- 再确认当前目录和目标路径

一个更稳妥的删除目录思路是：

1. 先 `cd` 进入目标目录
2. 用 `rm -r *` 删除内部内容
3. 确认目录已空
4. 回到上一级后再 `rmdir`

这比直接在外面敲一条大而危险的 `rm -rf some_dir` 要稳很多。

### 3. 查看文件内容

做科学计算时，你每天都在看输入文件和输出文件。

| 命令 | 全称 | 主要作用 |
| --- | --- | --- |
| `cat` | concatenate | 直接输出文件内容，适合小文件 |
| `less` | less | 分页查看文件，适合大文件 |
| `head` | head | 查看文件开头几行 |
| `tail` | tail | 查看文件最后几行，或持续追踪新输出 |

例如：

```bash
cat INCAR          # 直接输出小文件内容
head -n 20 OUTCAR  # 查看前 20 行
tail -n 20 OUTCAR  # 查看后 20 行
less OUTCAR        # 翻页查看大文件
```

简单理解：

- 小文件用 `cat`
- 大文件优先用 `less`
- 看文件开头用 `head`
- 看文件结尾和最新输出用 `tail`

还有一个很实用的用法：

```bash
tail -n 1 */result # 查看多个 result 文件的最后一行
```

它可以直接查看当前目录下一批结果文件的最后一行，在批量计算里很常见。

### 4. 文件传输

本地电脑和服务器是不能直接互相读取的对方的文件的，所以有时候需要把计算任务上传到服务器，然后把计算完的数据下载到本地做数据分析处理。

#### `scp`

```bash
# 从本地传文件到服务器
scp local.txt alias:~/work/        # 上传单个文件

# 从服务器传文件到本地
scp alias:~/work/result.txt ./     # 下载单个文件到当前目录

# 传目录时加 -r
scp -r local_dir alias:~/work/     # 上传整个目录
```

观察这个命令的结构其实很重要：

```bash
scp <source> <destination> # 从源路径复制到目标路径
```

这和本地复制命令 `cp` 很像，只是 `scp` 可以跨机器传输。

#### `rsync`

```bash
rsync -avh local_dir/ alias:~/work/local_dir/ # 本地同步目录到服务器
rsync -avh alias:~/work/results/ ./results/   # 服务器同步目录到本地
```

和 `scp` 相比，`rsync` 更像“同步”，不是单纯复制。

常见用法：

```bash
# 本地同步到服务器
rsync -avh local_dir/ alias:~/work/local_dir/ # 同步整个目录到服务器

# 服务器同步到本地
rsync -avh alias:~/work/results/ ./results/   # 同步整个目录到本地

# 只同步某类文件
rsync -avh *.txt alias:~/work/                # 同步当前目录下所有 txt 文件
```

其中常见参数：

- `-a`：归档模式，尽量保留文件信息
- `-v`：显示过程
- `-h`：人类可读的文件大小

`rsync` 有一个很重要的特点：它会比较源和目标之间的差异，所以非常适合反复同步同一个目录。

这意味着如果你经常做这些事情：

- 本地改脚本，再同步到服务器
- 服务器跑完结果，再同步回本地
- 中途中断后重新传输

那 `rsync` 往往会比 `scp` 更省时间，也更稳。

简单区分：

- 临时传一个文件，用 `scp`
- 需要同步很多文件、重复同步、希望更稳一点，优先用 `rsync`

#### 一个很实际的问题

`scp` 虽然不难，但实际使用起来：你在本地终端里，不一定记得服务器上的完整路径；你在服务器终端里，也不方便直接知道本地路径。你每次都得输入一串很长的目录名，非常麻烦。能不能偷懒？
能当然能！

我自己使用方法则是在服务器上准备一个 `put`：

```bash
#!/bin/bash
echo "scp -r alias:$PWD/$1 ." | pbcopy # 生成下载命令并复制到本地剪贴板
```

如果你是第一次看到这种脚本格式，可以先这样理解：

- `#!/bin/bash`：表示这个脚本用 `bash` 来执行
- `echo`：把一段文本输出出来
- `|`：管道，表示把前一个命令的输出交给后一个命令

所以这一行命令本质上是在做一件事：先生成一条 `scp` 命令，再把这条命令交给 `pbcopy`，复制到剪贴板里。

这个脚本做的事很简单：

- 读取你当前服务器目录 `PWD`
- 拼出一条下载命令
- 复制到本地剪贴板

然后你回到本地终端，直接粘贴执行就行。

这里的 `pbcopy` 是 macOS 自带的剪贴板命令。如果是 Linux Windows 系统可以基于 `OSC 52` 做一个简单的脚本：

```bash
#!/bin/bash
CMD="scp -r alias:$PWD/$1 ."                         # 先生成要执行的 scp 命令
ENCODED=$(printf "%s" "$CMD" | base64 | tr -d '\n') # 将命令做 base64 编码并去掉换行
printf '\e]52;c;%s\a' "$ENCODED"                     # 通过 OSC 52 协议把内容发送到本地剪贴板
```

这里：

- `CMD=...`：先把整条命令存到一个变量里
- `ENCODED=...`：把这条命令编码成终端可安全传输的格式
- `printf '\e]52;c;%s\a' ...`：通过 `OSC 52` 协议请求终端写入本地剪贴板

或者

```python
#!/usr/bin/env python
import sys      # 读取标准输入
import base64   # 把文本编码成终端可识别的剪贴板格式

buffer = []     # 保存输入内容
for line in sys.stdin:
    buffer.append(line)  # 逐行收集输入

content = ''.join(buffer).strip()  # 拼成完整字符串并去掉首尾空白
print('\033]52;c;' + base64.b64encode(content.encode()).decode() + '\007')  # 发送到本地剪贴板
```

这个方法的好处是不依赖服务器上有没有 `pbcopy`。只要你的本地终端支持 `OSC 52`，就可以直接把远程输出送回本地剪贴板。
记得打开终端工具允许写入剪贴板的功能。

这个思路的重点不在于“必须这样做”，而在于你要开始意识到：

- 命令行不只是手动敲命令
- 很多高频动作都可以给自己做一层小封装

同理，在本地你也可以准备一个简单脚本，用来把文件上传到服务器：

```bash
#!/bin/zsh
scp -r $1 alias:~ # 将本地文件或目录上传到服务器家目录
```

先把文件传到服务器家目录，再在服务器里用 `mv` 挪到目标位置。这个方式不一定是最优雅的，但对日常使用很实用，也足够稳。

最后补充一句：文件传输当然也有很多图形化界面工具，比如一些 SSH 客户端自带的文件面板、SFTP 文件浏览器之类。常见的 Windows 工具包括 [Xshell](https://www.netsarang.com/en/xshell/) 和 [MobaXterm](https://mobaxterm.mobatek.net/)。它们用起来更直观，但底层本质上还是这些远程传输协议和命令的封装。

另外还有一些在服务器环境里常见的工具，比如 [`lrzsz`](https://ohse.de/uwe/software/lrzsz.html)（`rz` / `sz`），也可以做文件上传下载。感兴趣的话可以自己探索，挑选适合自己的工具，还是那句话去跟 LLM 聊聊会很有收获。

## 入门检查清单

如果你现在刚入组，我建议你先把下面这些事做到能独立完成：

1. 登录服务器并配置一个 `ssh` 别名
2. 用 `ls`、`cd`、`cp`、`mv`、`rm` 整理目录
3. 用 `head`、`tail`、`less` 查看输入和输出文件
4. 用 `scp` 或 `rsync` 完成一次最基本的上传或下载
5. 知道本地终端和服务器终端不是一回事
