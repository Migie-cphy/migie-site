# 1.7 搜索、日志和报错排查

目标：不要再从头到尾硬读输出文件，而是先搜关键词，再看上下文。

## 1. 为什么要先搜再看

输出文件很长时，最慢的做法就是从头读到尾。

更实用的顺序通常是：

1. 先搜 `error`、`warning`、`failed`
2. 再看最后几十行
3. 再回头看相关上下文

## 2. `grep`

```bash
grep "pattern" file    # 在文件中搜索字符串
grep -r "pattern" dir  # 递归搜索目录中的文件
grep -i error job.out   # 忽略大小写搜索 error
grep -i warning OUTCAR  # 忽略大小写搜索 warning
```

先把它理解成“文本搜索器”就够了。

## 3. `tail`

```bash
tail -n 50 OUTCAR # 查看最后 50 行
tail -f job.out   # 实时追踪日志新输出
```

`tail -f` 很适合边跑边看日志。

## 4. 一个最常见的排查顺序

```bash
grep -i error OUTCAR # 先搜索报错关键词
tail -n 50 OUTCAR    # 再看最后 50 行上下文
```

这套顺序很简单，但已经能解决很多“程序到底跑没跑起来”的问题。

## 5. 这一节最少要做到什么

1. 会用 `grep` 搜一个关键词
2. 会用 `tail -n` 看文件结尾
3. 会用 `tail -f` 追踪日志
4. 知道先搜关键词，再看上下文
